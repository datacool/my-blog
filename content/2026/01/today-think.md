# 오늘의 일기

오늘은 정적 블로그 생성기를 처음부터 만들어보는 프로젝트를 시작했습니다.

## 무엇을 했나

- Node.js 기반 커스텀 SSG(Static Site Generator) 구축
- 마크다운 파일을 HTML로 변환하는 빌드 파이프라인 구현
- GitHub Pages 배포를 위한 정적 파일 생성 구조 설계
- 다크 테마 기반의 아름다운 블로그 디자인 적용

처음에는 복잡할 줄 알았는데, `gray-matter`와 `markdown-it` 같은 라이브러리를 활용하니 생각보다 빠르게 구현할 수 있었습니다.

## 배운 점

1. **프론트매터(Frontmatter)**의 활용  
   YAML 형식으로 메타데이터를 관리하면, 빌드 시 자동으로 카테고리와 태그를 분류할 수 있다는 걸 배웠습니다.

2. **정적 사이트 생성의 장점**  
   - 서버 비용 없음
   - 빠른 로딩 속도
   - SEO 친화적
   - GitHub Pages 같은 무료 호스팅 활용 가능

3. **템플릿 기반 렌더링**  
   간단한 문자열 치환만으로도 충분히 강력한 정적 페이지를 만들 수 있다는 걸 깨달았습니다.
